# App Factory Research - Compile, Host, and Deploy Applications

## Research Summary

### 1. WebContainer API (StackBlitz)

**What it does:**
- Browser-based runtime for executing Node.js applications
- Runs entirely client-side in browser tab
- Spins up dev environment in milliseconds
- Works offline

**Pricing:**
- Personal (Free): $0/month
  - Non-commercial usage with attribution
  - Up to 25,000 API sessions per month
- Enterprise: Contact sales for commercial use

**Key Features:**
- Native Node.js inside browser
- Run Webpack, Vite, and other toolchains
- Everything contained in browser tab (secure)
- Fast - milliseconds to boot

**Limitations:**
- Non-commercial requires attribution
- 25,000 sessions/month on free tier
- Commercial use requires enterprise license

### 2. Alternative Approaches for Zero-Cost

#### Option A: Server-Side Build System
Build apps on the Render.com server using subprocess:
- Use Python subprocess to run npm/yarn build
- Store built artifacts in temp directories
- Serve via static file endpoint
- Pros: Full control, no external dependencies
- Cons: Limited by Render free tier resources

#### Option B: Cloudflare Workers + D1
- Use Cloudflare Workers for compute
- D1 for database
- R2 for file storage
- Pros: Generous free tier, global edge
- Cons: Limited execution time (10ms CPU on free)

#### Option C: GitHub Actions for Build
- Push code to GitHub repo
- Use GitHub Actions to build
- Deploy to GitHub Pages or Cloudflare Pages
- Pros: Free, unlimited builds for public repos
- Cons: Requires GitHub integration

### 3. GitHub API for Repo Creation

**Endpoint:** POST /user/repos
**Authentication:** Personal Access Token or OAuth

```python
import requests

headers = {
    "Authorization": f"token {GITHUB_TOKEN}",
    "Accept": "application/vnd.github.v3+json"
}

data = {
    "name": "my-new-app",
    "description": "Generated by Infinity Worker",
    "private": True,
    "auto_init": True
}

response = requests.post(
    "https://api.github.com/user/repos",
    headers=headers,
    json=data
)
```

### 5. Implementation Plan

**Phase 1: Build System**
- Add subprocess-based build for Node.js projects
- Support npm, yarn, pnpm
- Capture build output and errors

**Phase 2: Internal Hosting**
- Create sandbox directories for each app
- Serve built apps via preview URLs
- Implement app lifecycle management

**Phase 3: GitHub Integration**
- Automated repo creation via GitHub API
- Push generated code to new repos
- Configure GitHub Actions for CI/CD

**Phase 4: Modular Disconnect**
- Export app as standalone package
- Include all dependencies and configs
- Generate deployment scripts for multiple platforms

**Phase 5: App Registry**
- Track all generated apps with PIDs
- Store metadata, versions, status
- Enable app management dashboard
